# 08 — UI Design System

## Philosophy

White-label neutral base with discipline-specific court art. Each discipline is immediately identifiable by its court, not by a theme color. The tenant's brand colors and logo inject into the shell — buttons, headers, scoreboards — while the court art remains discipline-true.

## Court art

Each discipline has an SVG component that renders the court at the top of the live scoring page. The court is decorative + functional: it shows player positions (where applicable) and current serve side.

### BeachCourt.tsx
- 16×8m court rendered as SVG
- Sandy texture (CSS radial gradients)
- Blue sky + ocean hint above far end
- Net centered
- Side labels: LEFT / RIGHT with serve arrow
- 2 player position dots per team

### GrassCourt.tsx
- Same dimensions as beach but with grass green gradient
- Subtle grass texture (CSS repeating pattern)
- Attack line not shown (beach rules)
- Player position dots (3–4 per team)

### IndoorCourt.tsx
- 18×9m court with hardwood floor texture
- Attack line at 3m clearly marked
- Position numbers 1–6 shown in substitution view
- Front zone / back zone color distinction
- Net with side bands

### LightCourt.tsx
- 12×6m court (smaller proportion)
- Light indoor surface (pale)
- Attack line at 2m
- Jump-serve restriction line at 1m behind end line (dashed)
- Position numbers for 4- or 5-player layout

### ActionBar states (all disciplines share this pattern)

```ts
type ActionBarPhase =
  | "BETWEEN_RALLIES"     // Main scoring: [Score A] [Start Rally] [Score B]
  | "RALLY_LIVE"          // Active: [Point A] [Replay] [Point B]
  | "TIMEOUT_ACTIVE"      // [End Timeout A] ████░░░░░░ [End Timeout B]
  | "TTO_ACTIVE"          // [End TTO]  (beach only)
  | "MEDICAL_ACTIVE"      // [End Medical Timeout]
  | "VCS_ACTIVE"          // [Upheld] [Rejected]
  | "LINEUP_PENDING"      // (indoor/light) waiting for both lineups
  | "SET_BREAK"           // Set break overlay (countdown)
  | "END_MATCH_MENU"      // confirm match end
  | "NOTE";               // note entry mode
```

**BETWEEN_RALLIES layout:**
```
[A TOs: 1] [TO A]  ◀ Score A  |  Score B ▶  [TO B]  [A subs: 5/6]
                   [   +1 A   ]  [start]  [   +1 B   ]
                   [  UNDO ×2 ]           [   NOTE    ]
```

For indoor: also shows [SUB A] [LINEUP A] [CHALLENGE A] etc.

### Two-tap score entry (existing pattern, kept)
- Tap once: arms score button
- Tap again within 5s: confirms
- Visual feedback: button pulses after first tap

### Two-tap UNDO
- Tap UNDO: arms for 5s
- Tap again: executes

### Keyboard shortcuts (scorer desktop)

| Key | Action |
|-----|--------|
| Space | Start rally / confirm point |
| A | +1 Team A (confirm point if RALLY_LIVE) |
| B | +1 Team B |
| R | Replay point |
| U then U | Undo (two-tap) |
| T then A / T then B | Timeout team A/B |
| N | Note entry |
| 1–6 | (Indoor) Sub slot number |

## Scoreboard display (`/t/[slug]/scoreboard/[matchId]`)

Full-screen read-only display. Auto-refreshes via Pusher subscription (no auth).

```
┌─────────────────────────────────────────────────────────┐
│              [Tenant Logo]  [Competition Name]           │
├────────────────────┬────────────────────────────────────┤
│                    │                                    │
│   TEAM A           │            TEAM B                  │
│   (full name)      │            (full name)             │
│                    │                                    │
│       2 3  ◀serve  │  serve▶  2 1                      │  Sets
│                    │                                    │
├────────────────────┼────────────────────────────────────┤
│                    │                                    │
│        21          │           18                       │  Current set score
│                    │                                    │
├────────────────────┼────────────────────────────────────┤
│  TOs: ●●           │  TOs: ●○                           │  Remaining TOs
│  Set 2             │  Time: 00:12:34                    │
└────────────────────┴────────────────────────────────────┘
```

Responsive: scales to 16:9 for TV/projector. Portrait mode for tablet/phone.

Scoreboard has three display modes (configurable per session):
- `SCORE_ONLY`: Big score, team names, set scores
- `SCORE_WITH_SETS`: Above + detailed set history
- `SCORE_WITH_ROTATION`: Above + current serving player name (indoor)

## Team tablet interface (`/t/[slug]/matches/[id]/team/[A|B]`)

Simplified, touch-optimized. Requires `TEAM_SCORER` session token.

│                                     │

│  Position 2:           [Player 3 ▼] │
│  Position 3:           [Player 5 ▼] │
│  Position 4:           [Player 2 ▼] │
│  Position 5:           [Player 4 ▼] │

│  Libero: [Designate ▼]             │
│                                     │

│  Team A  |  Set 2  |  Score: 18-16  │

│                                     │

Requests appear as notifications on the scorer's screen with [Approve] / [Deny] buttons.

### Notification on scorer screen (when team tablet sends request)
```
🔔 TEAM A requests TIMEOUT
   [Approve] [Deny]           ← must act before rally starts
```

## Set break overlay

When a set ends, an overlay appears on the scorer screen with:
- Set result (A 21 — B 18)
- Match score (Sets: 1-0)
- 60-second countdown before next set
- [Start Next Set] button (skip countdown)
- [Confirm Match End] if match is over

## Discipline configuration overlay

Admin can toggle features per competition:
- On/Off toggles for: Rotation, Lineup Entry, Server Tracking, TTO, Serve Clock, VCS, Team Tablets
- All changes take effect immediately for future matches in that competition
